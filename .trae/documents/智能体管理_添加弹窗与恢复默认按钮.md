# 智能体管理：添加时弹出人设编辑 + 恢复默认按钮

## 摘要

在智能体管理面板（`AgentPanel.tsx`）做两件事：

1. 点击「添加」按钮后，立即在屏幕中央弹出该新智能体的人设提示词编辑 Modal（复用现有 `PersonaEditorModal`）。用户取消时保留空白智能体（不回滚）。
2. 在面板头部新增「恢复默认」按钮（注意使用图标按钮，不要出现汉字），点击后弹出可复用的确认对话框，确认后调用 store 的 `resetToDefaults()`。

同时新建一个可复用的 `ConfirmDialog` 组件，供「恢复默认」及未来所有确认场景使用，视觉风格与现有 `PersonaEditorModal` 完全一致。

## 提议改动

### 改动 1：新建可复用确认对话框组件 `ConfirmDialog`

**文件**：`src/components/ConfirmDialog.tsx`（新建）

**目的**：提供项目内统一的确认弹窗，替代 `window.confirm`，风格与 `PersonaEditorModal` 完全一致。

**Props 设计**：

```tsx
interface ConfirmDialogProps {
  title: string;                  // 标题
  message: string;                // 正文说明
  confirmText?: string;           // 确认按钮文案，默认 "确认"
  cancelText?: string;            // 取消按钮文案，默认 "取消"
  variant?: 'primary' | 'danger'; // 确认按钮样式，默认 'primary'
  onConfirm: () => void;          // 确认回调
  onCancel: () => void;           // 取消/关闭回调
}
```

**实现要点**：

* 结构完全照搬 `PersonaEditorModal`：Overlay + 居中容器 + Header（标题 + X）+ Body（message 文本）+ Footer（取消 + 确认）

* 内层卡片宽度 `max-w-md`（比人设编辑器的 `max-w-lg` 略窄，因内容更简单）

* `variant: 'danger'` 时，确认按钮 `backgroundColor: var(--destructive)` `color: #fff`

* `variant: 'primary'` 时，确认按钮 `backgroundColor: var(--primary)` `color: #fff`

* 点击 overlay 或 X 或取消按钮均触发 `onCancel`

* 点击确认按钮触发 `onConfirm`

* 不内部管理开关状态，由调用方控制挂载/卸载（与 `PersonaEditorModal` 一致）

### 改动 2：`AgentPanel.tsx` — 添加智能体时立即弹出人设编辑 Modal

**文件**：[src/components/AgentPanel.tsx](file:///d:/python/WeDebate-model_debatement_room/src/components/AgentPanel.tsx)

**改动位置**：`AgentPanel` 组件（L257-L374）

**实现要点**：

1. 新增 state：`const [newAgentForEdit, setNewAgentForEdit] = useState<AgentConfig | null>(null);`
2. 修改 `handleAddAgent`（L267-L277）：创建 `newAgent` 后，除了 `addAgent(newAgent)`，再调用 `setNewAgentForEdit(newAgent)` 触发弹窗。
3. 在 `AgentPanel` 的 JSX 末尾（`</>` 前）追加条件渲染：

   ```tsx
   {newAgentForEdit && (
     <PersonaEditorModal
       agent={newAgentForEdit}
       onSave={(personaPrompt) => {
         updateAgent(newAgentForEdit.id, { personaPrompt });
         setNewAgentForEdit(null);
       }}
       onCancel={() => setNewAgentForEdit(null)}
     />
   )}
   ```
4. 需在 `AgentPanel` 中从 store 取 `updateAgent`：`const updateAgent = useAgentStore((s) => s.updateAgent);`

**取消行为**：`onCancel` 仅 `setNewAgentForEdit(null)`，**不删除**刚创建的空白智能体（符合用户选择）。用户可在列表中继续编辑或删除。

**注意**：`PersonaEditorModal` 内部用 `useState(agent.personaPrompt)` 初始化 value，新智能体 `personaPrompt: ''` 可正常工作。Modal 标题 `编辑人设提示词 — 新智能体` 也正确显示。

## 假设与决策

| 项                | 决策                                             | 依据                                  |
| ---------------- | ---------------------------------------------- | ----------------------------------- |
| 取消弹窗后是否删除新智能体    | 不删除，保留空白智能体                                    | 用户明确选择                              |
| 恢复默认按钮位置         | 头部「添加」按钮左侧                                     | 用户选择「头部按钮」；放左侧用次要样式避免抢主操作焦点         |
| 恢复默认按钮样式         | 次要样式（透明底 + 边框）                                 | 与现有次按钮（如「取消」）风格一致，且区分于主操作「添加」       |
| 是否需要二次确认         | 需要，用 `ConfirmDialog`                           | 用户明确选择；破坏性操作（覆盖所有自定义智能体）            |
| 确认弹窗组件           | 新建可复用 `ConfirmDialog`                          | 用户要求组件复用                            |
| ConfirmDialog 宽度 | `max-w-md`                                     | 比人设编辑器 `max-w-lg` 略窄，符合确认场景内容更简单的语义 |
| z-index 层级       | `ConfirmDialog` 与 `PersonaEditorModal` 同为 z-50 | 后渲染者在上；二者不会同时出现（恢复默认时面板内无编辑 Modal）  |
| 不引入新依赖           | 仅用 lucide-react 的 `RotateCcw`                  | 项目已有 lucide-react                   |
