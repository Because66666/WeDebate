# Checklist

## 架构与代码质量
- [x] 项目目录结构清晰，分层明确（components / stores / services / types / utils / prompts）
- [x] 依赖方向正确：components → stores → services → types
- [x] TypeScript类型定义完整，无any类型滥用
- [x] 各模块职责单一，功能解耦

## 提示词文件组织
- [x] 基础提示词独立为单独文件（`src/prompts/base.ts`），所有智能体共享
- [x] 每个智能体的人物扮演提示词独立文件（`src/prompts/agents/`文件夹）
- [x] 新增智能体只需添加一个提示词文件，无需修改基础提示词

## 智能体上下文构建
- [x] 每个智能体拥有独立的系统提示词（基础 + 人物扮演）
- [x] 其他智能体输出和用户消息以user角色注入，格式为"角色名:内容"
- [x] 多智能体上下文互不干扰
- [x] 上下文消息格式符合OpenAI API规范

## 发言调度机制
- [x] 智能体按次序依次发言
- [x] 用户中途插嘴时，当前智能体完成输出后暂停后续调度
- [x] 用户发言后，发言次序回退1位（用户发言前的智能体插队为首位）
- [x] 无用户插嘴时，持续3个完整轮次后自动停止
- [x] 调度状态正确管理（当前发言者、轮次计数、插队状态）

## 流式输出
- [x] 使用原生fetch + ReadableStream解析SSE
- [x] 流式内容实时追加显示
- [x] 正确处理 data: [DONE] 流结束标识
- [x] 每个智能体消息独立显示，互不干扰
- [x] 流式渲染时进行性能优化（节流渲染，避免页面卡顿）
- [x] 流式传输错误有友好提示

## Markdown渲染

- [x] 支持有序/无序列表
- [x] 支持代码块（语法高亮）

- [x] 支持分割线
