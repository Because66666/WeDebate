# WeDebate：Model Debate Room · 模型辩论室

> **多智能体群聊讨论平台 — 让多个 AI 同台辩论，碰撞思维火花**

---

## 创意名称 + 创意介绍

### 项目名称

**Model Debate Room（模型辩论室）**

### 想解决什么问题

单一 AI 助手的回答往往受限于其固定人设和视角，用户难以在同一对话中获得多元、对立的观点碰撞，导致思考维度单一、决策视角受限。

### 为什么会想到做这个

兼听则明，偏信则暗。在现实讨论中，不同背景的人各抒己见才能让问题越辩越明。受此启发，我们想让多个具备不同专业人设的 AI 智能体在同一个"圆桌"上展开讨论，用户作为主持人参与其中，从而获得更立体、更全面的认知。

### 大概是什么产品

**纯浏览器端的 Web 应用**（SPA），用户可与多个预设人设的 AI 智能体进行多轮群聊讨论，每个智能体独立发言、彼此感知，形成真正的多角色辩论与协作体验。

---

## 目标用户及痛点

### 面向哪些用户

- **知识工作者**：需要多角度分析问题的产品经理、咨询师、研究员、决策者
- **AI 探索者**：对多智能体协作和大语言模型能力边界感兴趣的开发者和科技爱好者
- **内容创作者**：需要灵感碰撞和观点对撞的写作者、媒体人、策划人员
- **教育学习者**：希望通过多视角讨论来深入理解复杂课题的学生和自学者

### 在什么场景下使用

- **决策辅助**：在做重要决策前，让科技顾问、金融顾问等不同角色分别分析利弊，获得多角度的建议和建议
- **头脑风暴**：围绕一个主题，让多个智能体从各自专业视角提出方案和见解
- **知识探究**：针对复杂问题，通过多轮辩论式讨论逐步深入，获得全方位的理解
- **思维模拟**：模拟多方利益相关者的立场和观点，预判不同决策的可能后果

### 当前痛点

| 痛点 | 说明 |
|------|------|
| **视角单一** | 传统 AI 聊天只能一对一对话，无法同时获得多个角度的分析 |
| **缺乏辩论感** | 单一 AI 的回答缺乏与其他观点的对撞和交锋，结论趋于保守 |
| **切换成本高** | 用户需要手动在不同对话间切换不同人设的 AI，无法在同一语境下横向对比 |
| **上下文割裂** | 不同 AI 各自为战，无法感知彼此的观点，难以形成连贯的群体讨论 |

## 价值与意义

### 效率提升

- **一站式多视角分析**：一次提问即可获得来自科技、金融等多个专业领域的独立分析与建议，无需分别提问、手动整合
- **多轮自动辩论调度**：系统自动编排智能体的发言顺序和轮次，用户只需关注讨论内容本身
- **流式实时呈现**：所有智能体的回答均为 SSE 流式输出，逐字显示，无需等待完整响应
- **即开即用**：纯前端架构，无需部署服务器，配置 API Key 即可使用

### 商业价值

- **低成本高灵活度**：仅需一个 OpenAI 兼容 API 即可驱动多个智能体并行讨论，边际成本几乎为零
- **可定制化智能体系统**：支持自由添加、编辑智能体人设，企业可快速搭建符合自身业务场景的多角色专家顾问团
- **数据本地化**：所有对话数据存储在浏览器 localStorage 中，不经过第三方服务器，满足数据隐私合规要求

## 产品特色

### 相比传统 AI 聊天的优势

| 特性 | 传统 AI 聊天 | Model Debate Room |
|------|-------------|-------------------|
| 同时对话的 AI 数量 | 1 个 | 任意多个（可自由启用/禁用） |
| 智能体人设 | 通用助手（固定不变） | 独立人设 + 专业背景 + 个性化语气，可自由创建任意角色 |
| 多角色感知 | 无 | 每位智能体可感知其他智能体的发言 |
| 讨论形式 | 一问一答 | 多轮辩论式群聊 |
| 用户角色 | 提问者 | 主持人 + 参与者 |
| 观点多样性 | 单一视角 | 多角度对撞分析 |
| 搜索能力 | 取决于平台 | 集成 DuckDuckGo 搜索（规划中） |

### 核心功能一览

- **多智能体群聊**：预设科技顾问、金融顾问等多位 AI 专家，支持自定义人设，也可自由添加任意角色（如法律顾问、心理分析师、历史学家等），获得更多专业视角
- **多轮自动调度**：系统自动编排发言顺序与轮次，形成连贯的群组讨论
- **用户插队机制**：讨论过程中可随时发送消息介入，引导对话方向
- **流式输出 + 滑动窗口动画**：逐 token 实时渲染，体验流畅自然
- **Markdown 结构化渲染**：完整支持代码块、表格、列表等富文本格式
- **对话历史管理**：自动保存所有对话，支持新建、切换、删除
- **智能标题生成**：书记官智能体自动为每场辩论生成概括性标题
- **深色/亮色主题**：适配不同使用环境
- **扁平设计 UI**：清晰、简洁、无干扰的界面

## 快速开始

### 前置要求

- Node.js 18+
- 一个 OpenAI 兼容的 API Key

### 安装与运行

```bash
# 安装依赖
npm install

# 启动开发服务器
npm run dev

# 构建生产版本
npm run build
```

### 配置

1. 启动后在浏览器中打开应用
2. 点击设置按钮，填写：
   - **API Endpoint**：你的 API 端点地址（默认兼容 OpenAI）
   - **API Key**：你的 API 密钥
   - **模型名称**：选择或输入要使用的模型
3. 选择一个对话或新建对话
4. 在输入框中发送消息，即可开启多智能体辩论

## 技术架构

```
src/
├── types/         # TypeScript 类型定义
├── stores/        # Zustand 状态管理（对话/智能体/设置/轮次）
├── services/      # 核心服务层
│   ├── chat-service.ts     # 聊天调度器
│   ├── turn-manager.ts     # 发言轮次调度
│   ├── context-builder.ts  # 智能体上下文构建
│   ├── openai.ts           # 流式 API 客户端
│   └── tools/              # 工具注册表与搜索工具
├── prompts/       # 智能体人设提示词
├── components/    # React UI 组件
└── App.tsx        # 根组件
```

**核心数据流**：用户发言 → 轮次调度器依次选定智能体 → 上下文构建器组装独立提示 → 流式 API 逐 token 输出 → 实时渲染到聊天面板 → 完成后推进至下一智能体 → 所有轮次结束 → 自动生成对话标题。
